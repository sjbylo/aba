Install & manage air-gapped OpenShift. 

  Aba makes it easier to install an OpenShift cluster - 'Cluster Zero' - into a fully or partially
  disconnected environment, either onto bare-metal, vSphere or ESXi. Because Aba uses the Agent-based
  installer there is no need to configure a load balancer, a bootstrap node or require DHCP.

Main commands: 
  aba                                   # Interactive mode.  Let Aba guide you through the process
  aba [-d mirror] install               # Install Quay (or connect to existing) mirror registry. See 'aba mirror --help'.
  aba [-d mirror] verify                # Verify connectivity with the mirror registry
  aba [-d mirror] sync                  # Copy images from the Internet into the mirror registry (m2m)
  aba [-d mirror] save|load             # Copy images from the Internet to disk (m2d) and/or load to mirror (d2m)
  aba bundle                            # Create a bundle archive file. See 'aba bundle --help'
  aba cluster                           # Install OpenShift cluster. See 'aba cluster --help'

Other commands:
  aba clean                             # Clean up all generated files
  aba [-d cli] download                 # Download all required CLI installation files only
  aba [-d cli] install                  # Download and install all the CLI binaries into ~/bin
  aba help                              # See help for all steps/targets

Options:
  -y                                    # Automatically accept the default answer to all prompts
  -d, --dir <path/to/dir>               # Run aba in the specified directory, or cd there and run without -d.
  -h, --help                            # Show this help information
  -S, --pull-secret <path/to/file>      # Location of your pull secret (json) file here
  -c, --channel <fast|stable|candidate> # Set the OpenShift installation channel, e.g. fast|f, stable|s, eus or candidate|c
  -v, --version <version>               # Set (x.y.z) or (x.y) OpenShift version, e.g. 4.19.20 or 4.19 or 'latest' or 'l' or 'previous' or 'p'
  -p, --platform <vmw|bm>               # Set the target platform, e.g. vmw (vCenter or ESX) or bm (bare-metal). This changes the install flow
  -b, --base-domain <domain>            # Set the OpenShift base domain, e.g. company.com
  -M, --machine-network <cidr>          # Set the OpenShift cluster's host/machine network address, e.g. 10.0.0.0/24
  -N, --dns <ip address(es)>            # Set one or more DNS IP address
  -R, --default-route <next hop ip>     # Set the default route of the internal network
  -T, --ntp <hostname(s) or IPs>        # Set the NTP hostname(s) or IP address(es) (recommended!)
  -O, --ops [<list of operators>]       # Add individual operators to your image set config (ISC) file (for oc-mirror)
  -P, --op-sets [<operator set list>]   # Add sets of operators to your ISC file, as defined by 'aba/templates/operator-set.*' files. 'all' is accepted
  -e, --editor <editor command>         # Set the editor to use, e.g. vi, emacs, pico, none...  'none' means manual editing of config files
  -a, --ask                             # Prompt user when needed
  -y, --noask                           # Do not prompt, assume default answers
  -o, --out <file|->                    # Output Install Bundle to destination, e.g. file or stdout (-). See 'aba bundle -h`
  -f, --force                           # Only used for 'aba reset --force'
  -V, --vmware                          # Apply this VMware config file (for govc)
  -D, --debug                           # Increase output
  -q, --quiet                           # Less output
      --info                            # Increase output (default)

Usage:
  aba <command> [options]

Use "aba <command> -h|--help" for more information about a given command
