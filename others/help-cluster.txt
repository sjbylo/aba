Install OpenShift

Usage (from root of aba directory):
  aba cluster --name <name> [options]                                # Create a cluster directory

Examples:
  aba cluster --name mycluster --type standard                       # Create the cluster dir and install OpenShift
  aba cluster --name mycompact --type compact --step agentconf       # Create agent-based config files and stop
  aba cluster --name sno --type sno --step iso                       # Create the agent-based iso and stop
  aba cluster --name sno --type sno --starting-ip 10.0.0.1           # Install SNO cluster, setting the first node's IP

Options:

  aba cluster                             # Create a cluster dir with cluster configuration (cluster.conf)
    -y                                    # Automatically accept the default answer to all prompts
    -n, --name <string>                   # The name of the cluster to create
    -t, --type <sno|compact|standard>     # Set type of cluster. Default is 'standard'.
    -s, --step <step>                     # Process up to this step or target <cluster.conf|agentconf|iso|mon>
                                          # See steps (targets) with "aba --dir mycluster help"
    -i, --starting-ip <ip>                # IP addr for the first cluster node (rendevous/node0)
        --api-vip <ip>                    # IP addr for cluster's API
        --ingress-vip <ip>                # IP addr for application ingress
    -I, --int-connection [<proxy|direct>] # Install from the public reg. or set to empty to use mirror
                                          # Below options ONLY for platform=vmw
        --mcpu <count>                    # Set the master cpu count in vCPU
        --mmem <count>                    # Set the master memory in GB
        --wcpu <count>                    # Set the worker cpu count in vCPU
        --wmem <count>                    # Set the worker memory in GB
        --data-disk [<size>]              # Size, in GB, of thin prov. 'data disk' or empty for no data disk

Usage (from a cluster directory):
  aba --cmd "oc command"                  # Run this 'oc command' on the current cluster
  aba ls                                  # List all cluster node VMs (for platform=vmw only)
  aba shutdown                            # Gracefully shut down the cluster (any platform)
  aba startup                             # Gracefully start up the cluster (any platform)
  aba help                                # See help for all steps/targets
