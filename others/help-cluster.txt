Install OpenShift

Usage (from root of aba directory):
  aba cluster --name <string> [options]                              # Create a cluster directory.

Examples:
  aba cluster --name mycluster --type standard                       # Create the cluster dir and install OpenShift.
  aba cluster --name mycompact --type compact --step agentconf       # Create agent-based config files and stop.
  aba cluster --name sno --type sno --step agentconf|iso|install     # Create cluster dir and process until 'step'.
  aba cluster --name sno --type sno --starting-ip 10.0.0.1           # Install SNO cluster, setting the first node's IP.
  aba cluster --name ocp -A 10.0.0.5 -G 10.0.0.6                     # Set VIPs for API and ingress endpoints.
  aba cluster --name mycluster --ports ens192 ens224                 # Specify network ports used for cluster nodes.

Options:

  aba cluster                             # Create a cluster dir with cluster configuration (cluster.conf).
    -d, --dir <path/to/dir>               # Run aba in the specified directory, or cd there and run without -d.
    -y                                    # Automatically accept the default answer to all prompts.
    -n, --name <string>                   # The name of the cluster to create.
    -t, --type <sno|compact|standard>     # Set type of cluster. Default is 'standard'.
    -s, --step <step>                     # Process up to this step or target <cluster.conf|agentconf|iso|mon>.
                                          # See steps (targets) with "aba --dir mycluster help".
    -i, --starting-ip <ip>                # IP address for the first cluster node (rendezvous/node0).
    -A, --api-vip <ip>                    # IP address for cluster's API.
    -G, --ingress-vip <ip>                # IP address for cluster application ingress.
    -M, --machine-network <cidr>          # CIDR of node machine network, such as 10.0.0.0/20.
    -g, --gateway-ip <ip>                 # Gateway IP or next hop address.
    -I, --int-connection [proxy|direct]   # Install from the public registry or set empty to use mirror.
        --ports [<string> ...]            # Space-separated list of network port names or empty to remove.
                                          # ports in <cluster name>/cluster.conf.

  # Below options ONLY for platform=vmw
        --mcpu <count>                    # Set master CPU count (vCPUs)
        --mmem <count>                    # Set master memory (GB)
        --wcpu <count>                    # Set worker CPU count (vCPUs)
        --wmem <count>                    # Set worker memory (GB)
        --data-disk-gb [<size>]           # Size (GB) of thin-provisioned 'data disk' or empty for none.
    -W, --num-workers <count>             # Set number of worker nodes.
        --num-masters <count>             # Set number of master/control-plane nodes.
        --vlan [<tag>]                    # Set VLAN tag or empty to clear.
        --host-prefix <bits>              # Pod subnet prefix per node (default: 23).
        --master-prefix <name>            # Node name prefix for masters (default: master).
        --worker-prefix <name>            # Node name prefix for workers (default: worker).
        --ssh-key [<path>]                # Path to SSH public key or empty to clear.
        --mirror-name <name>              # Mirror directory this cluster uses (default: mirror).
                                          # Set to match a named mirror dir (e.g. enclave1).

Usage (from a cluster directory):
  aba run --cmd "oc command"              # Run this 'oc command' on the current cluster.
  aba unstick                             # Bounce pods that aren't fully ready (recovery for stuck installs).
  aba upgrade --to <version>              # Upgrade cluster to target OCP version (disconnected).
  aba shutdown                            # Gracefully shut down the cluster (any platform).
  aba startup                             # Gracefully start up the cluster (any platform).

Upgrade options:
  aba [-d <cluster>] upgrade
      --to <version>                      # Target OCP version (e.g. 4.19.28). Falls back to mirror.conf:ocp_version_target.
      --force                             # Pass --force to oc adm upgrade
      --dry-run                           # Show what would happen without executing
      --skip-day2                         # Don't auto-run day2 even if IDMS is missing

Usage for platform=vmw:
  aba ls                                  # List all cluster node VMs
  aba refresh                             # Re-create the VMs
  aba create [--start]                    # Create the VMs if they don't exist
  aba stop                                # Shut down the VMs 
  aba start                               # Start the VMs 
  aba kill                                # Power off the VMs
