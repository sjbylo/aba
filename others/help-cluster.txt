Install OpenShift

Usage (from root of aba directory):
  aba cluster --name <string> [options]                              # Create a cluster directory

Examples:
  aba cluster --name mycluster --type standard                       # Create the cluster dir and install OpenShift
  aba cluster --name mycompact --type compact --step agentconf       # Create agent-based config files and stop
  aba cluster --name sno --type sno --step agentconf|iso|all         # Create cluster dir and process until 'step'
  aba cluster --name sno --type sno --starting-ip 10.0.0.1           # Install SNO cluster, setting the first node's IP
  aba cluster --name ocp --api-vip 10.0.0.5 --ingress-vip 10.0.0.6   # Set VIPs for API and ingress endpoints
  aba cluster --name mycluster --ports ens192 ens224                 # Specify network ports used for cluster nodes

Options:

  aba cluster                             # Create a cluster dir with cluster configuration (cluster.conf)
    -d, --dir <path/to/dir>               # Run aba in the specified directory, or cd there and run without -d.
    -y                                    # Automatically accept the default answer to all prompts
    -n, --name <string>                   # The name of the cluster to create
    -t, --type <sno|compact|standard>     # Set type of cluster. Default is 'standard'.
    -s, --step <step>                     # Process up to this step or target <cluster.conf|agentconf|iso|mon>
                                          # See steps (targets) with "aba --dir mycluster help"
    -i, --starting-ip <ip>                # IP address for the first cluster node (rendezvous/node0)
        --api-vip <ip>                    # IP address for cluster's API
        --ingress-vip <ip>                # IP address for application ingress
    -I, --int-connection [proxy|direct]   # Install from the public registry or set empty to use mirror
        --ports [<string> ...]            # Space-separated list of network port names or empty to remove
                                          # ports in <cluster name>/cluster.conf

  # Below options ONLY for platform=vmw
        --mcpu <count>                    # Set master CPU count (vCPUs)
        --mmem <count>                    # Set master memory (GB)
        --wcpu <count>                    # Set worker CPU count (vCPUs)
        --wmem <count>                    # Set worker memory (GB)
        --data-disk [<size>]              # Size (GB) of thin-provisioned 'data disk' or empty for none

Usage (from a cluster directory):
  aba --cmd "oc command"                  # Run this 'oc command' on the current cluster
  aba ls                                  # List all cluster node VMs (for platform=vmw only)
  aba shutdown                            # Gracefully shut down the cluster (any platform)
  aba startup                             # Gracefully start up the cluster (any platform)
  aba help                                # See help for all steps/targets
