Set up and remove infrastructure services (DNS and NTP)

Usage:
  aba setup dns [options]               # Set up dnsmasq as local DNS resolver
  aba setup ntp [options]               # Set up chronyd as NTP server for the cluster network
  aba remove dns [-y]                   # Remove ABA's dnsmasq configuration and restore original DNS
  aba remove ntp [-y]                   # Remove NTP server 'allow' configuration from chronyd

Options (setup dns):
  -y, --yes                             # Answer yes to all prompts
      --bastion-ip <IP>                 # Override auto-detected bastion IP address
      --upstream <IP>                   # Override auto-detected upstream DNS server

Options (setup ntp):
  -y, --yes                             # Answer yes to all prompts
      --bastion-ip <IP>                 # Override auto-detected bastion IP address
      --allow-network <CIDR>            # Override auto-detected allowed network (e.g. 10.0.0.0/16)

Options (remove):
  -y, --yes                             # Answer yes to all prompts

  After 'aba setup dns' is run, aba.conf is updated with dns_servers and DNS records for
  the mirror registry and each cluster are auto-managed during install and delete.
  Auto-management is a no-op if 'aba setup dns' was never run.

  After 'aba setup ntp' is run, aba.conf is updated with ntp_servers.

Examples:
  aba setup dns                               # Auto-detect everything
  aba setup dns --upstream 10.0.1.8           # Use specific upstream DNS
  aba setup ntp                               # Auto-detect network from bastion IP
  aba setup ntp --allow-network 10.0.0.0/16   # Allow a specific network
  aba remove dns -y                           # Non-interactive removal
  aba remove ntp -y                           # Non-interactive removal
